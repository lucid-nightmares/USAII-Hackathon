# Role 6: Red Team Skeptic

Based on the current product and the previous role outputs.

## 10 remaining realism holes
1. The reveal boundary is still mostly illustrative until a real reveal mechanic exists.
2. Incident state is not yet persisted as a first-class model.
3. The route state is descriptive, not yet deeply tied to incident actions.
4. Confidence is visible but not yet clearly changing guardian permissions.
5. There is no single transparency trail for sensitive data access.
6. Integrity issues do not yet visibly reduce trust in monitoring coverage.
7. The demo can still look scripted because every child profile starts in a known shape.
8. Follow-up outcomes are not explicit enough after a parent acts.
9. The product still risks looking like a school tool reskinned for families if the trust boundary is not operationally stronger.
10. Some controls still look like configuration theater unless they influence current incidents.

## 5 overclaim risks
1. Claiming local-first privacy without proving reveal containment.
2. Claiming calm intervention if medium-confidence incidents still allow restrictive actions too easily.
3. Claiming auditability when access-to-content events are not logged.
4. Claiming integrity awareness when degraded monitoring does not change UI trust signals.
5. Claiming product realism if policy changes never affect active incidents.

## 5 trust-breaking moments
1. A parent can see more than the copy suggests without an explicit boundary crossing.
2. A low-confidence incident looks visually too similar to a critical one.
3. A restrictive action is available without explaining why it is justified.
4. The system says monitoring is healthy when a coverage component is degraded.
5. The action history tells what a parent did but not whether it exposed extra child data.

## 5 fixes required before final implementation
1. Add guarded reveal with reason and audit trail.
2. Add incident state machine and route-state updates.
3. Add dignity-mode gating for medium-confidence incidents.
4. Add integrity degradation that visibly affects coverage language and trust.
5. Expand the audit log into a transparency log for reveals, escalations, approvals, and rule changes.
